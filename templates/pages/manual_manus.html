1. Brand Foundation

Essência da Marca

MIKE é a disciplina implacável que transforma esforço em progresso mensurável.

Personalidade

  • Objetivo: Direto ao ponto, sem floreios.
  • Preciso: Cada dado é tratado com exatidão.
  • Motivador: Inspira a busca contínua por melhoria.
  • Disciplinado: Reflete a disciplina necessária para o treinamento Heavy Duty.

Princípios de Design

Foco no Essencial

Eliminar qualquer elemento que não contribua diretamente para o registro ou análise do treino.

Clareza e Legibilidade

As informações mais importantes devem ser imediatamente compreensíveis.

Feedback Instantâneo

Cada interação deve gerar uma resposta clara e imediata.

Consistência Previsível

Elementos de interface devem ser consistentes em todo o app.

8. Do / Don't

DO

Use o MIKE Accent para destacar ações primárias e progresso. Mantém o foco e motiva o usuário.

DON'T

NÃO use cores de erro para indicar sucesso. Isso cria confusão e ansiedade.

DO

Mantenha contraste mínimo de 4.5:1 para texto. Garante acessibilidade e legibilidade.

DON'T

NÃO use cores muito claras em fundos claros. Compromete a legibilidade.

DO

Use Oswald para títulos e números grandes. Cria impacto e eficiência visual.

DON'T

NÃO use múltiplas fontes diferentes. Compromete a coesão visual.

9. Checklist de Implementação

Use este checklist para garantir que o design system está sendo implementado corretamente em todo o projeto.

Cores

Definir variáveis CSS para todas as cores (light e dark mode)
Validar contraste de todas as combinações de cor/texto (WCAG AA)
Implementar toggle de dark mode com localStorage

Tipografia

Importar Google Fonts (Montserrat, Oswald, Roboto Mono)
Definir escala tipográfica (H1-H6, body, small, caption)
Aplicar line-heights corretos para legibilidade

Componentes

Implementar botões (primary, secondary, ghost, danger)
Criar inputs com estados (focus, error, disabled)
Implementar cards com sombras e bordas
Criar badges para status (success, error, warning, info)
Implementar tabelas com alternância de linhas

Acessibilidade

Testar com ferramentas de contraste (ex: WebAIM)
Garantir focus rings visíveis em todos os elementos interativos
Testar navegação por teclado
Validar com screen readers

Responsividade

Testar em breakpoints (mobile, tablet, desktop)
Garantir que ícones e imagens escalam corretamente
Validar legibilidade de números em telas pequenas

Performance

Otimizar fontes do Google Fonts (usar font-display: swap)
Minificar CSS e JavaScript
Testar performance em conexões lentas
